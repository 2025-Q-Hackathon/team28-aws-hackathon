**IGRUS : Love Q - AI 연애 답변 도우미** 💕

## **어플리케이션 개요**

Love Q는 사용자의 말투와 감정을 학습하여 개인화된 연애 답변을 제공하는 AI 서비스입니다. 연애 대화에서 어떻게 답변해야 할지 고민하는 사람들을 위해, 상대방의 성격과 소통 스타일을 분석하여 최적의 답변을 제안합니다.

**핵심 가치:**
- 🎯 **개인화**: 사용자의 말투와 상대방 특성을 종합 분석
- 🧠 **AI 기반**: AWS Bedrock과 Comprehend를 활용한 고도화된 분석
- 💕 **관계 개선**: 상대방에게 맞는 소통 방식 제안
- 📱 **편의성**: 간단한 입력으로 즉시 답변 생성

## **주요 기능**

### 1. 말투 분석 시스템
<img width="1708" height="840" alt="image" src="https://github.com/user-attachments/assets/0ffde102-8700-4775-99fc-c714b6add5b5" />


- **존댓말/반말 비율 분석**: 문장 종결어 패턴을 통한 격식도 측정
- **이모티콘 사용 빈도**: 감정 표현 스타일 파악
- **평균 메시지 길이**: 대화 선호도 분석
- **성격 특성 추출**: AI 기반 개인 성향 분석

```javascript
// 말투 분석 예시
{
  "formal_ratio": 0.3,      // 30% 존댓말 사용
  "emoji_ratio": 0.4,       // 40% 이모티콘 포함
  "avg_length": 25,         // 평균 25자 메시지
  "personality": "활발하고 친근한 스타일"
}
```

### 2. 상대방 프로필 분석 ✨
- **성격 특성 분석**: 내향/외향, 직접/간접 소통 스타일
- **관심사 파악**: 취미, 선호하는 대화 주제
- **소통 패턴**: 상대방이 선호하는 대화 방식
- **호환성 점수**: 사용자와의 궁합 분석

### 3. 개인화 답변 생성
- **안전형**: 무난하고 안전한 답변 (위험도 1-2)
- **균형형**: 적당한 관심 표현 (위험도 2-3)  
- **대담형**: 적극적인 호감 표현 (위험도 3-5)
- **상황별 최적화**: 상대방 성향에 맞는 맞춤 전략

### 4. 감정 분석 (AWS Comprehend)
- **실시간 감정 상태 분석**: POSITIVE/NEGATIVE/NEUTRAL/MIXED
- **감정 강도 측정**: 신뢰도 점수 기반
- **핵심 키워드 추출**: 중요한 감정 표현 식별
- **개체명 인식**: 인물, 장소, 시간 정보 추출

### 5. 사용자 인증 & 개인화
- **AWS Cognito 기반 안전한 로그인**
- **대화 히스토리 관리**: 개인별 학습 데이터 축적
- **성공률 추적**: 답변 효과 분석
- **실시간 대시보드**: 사용 통계 및 성과 지표

## **리소스 배포하기**

### **시스템 아키텍처**
```
Frontend (Next.js) → Amplify Auth → API Gateway → Lambda Functions
                         ↓              ↓              ↓
                    Cognito        Comprehend      DSQL/S3
                                       ↓              ↓
                                   Bedrock (AI)   대화기록
```

### **배포 단계**

#### 1. 백엔드 배포 (서버리스 스택)
```bash
# 1. CloudFormation 스택 배포
./deploy-backend.sh dev us-east-1

# 배포되는 AWS 리소스:
# - Lambda Functions (6개): 말투분석, 감정분석, 답변생성 등
# - API Gateway: RESTful API 엔드포인트
# - Cognito User Pool: 사용자 인증
# - DSQL Database: 사용자 데이터 저장
# - S3 Bucket: 파일 업로드 (7일 자동 삭제)
# - IAM Roles: 최소 권한 원칙
```

#### 2. 환경변수 설정
```bash
# 2. 프론트엔드 환경변수 자동 생성
./setup-env.sh dev us-east-1

# 생성되는 환경변수:
# - API Gateway URL
# - Cognito User Pool ID
# - Cognito Client ID
# - AWS Region
```

#### 3. 프론트엔드 배포 (AWS Amplify)
```bash
# 방법 1: AWS 콘솔 (추천)
# 1. AWS Amplify 콘솔에서 새 앱 생성
# 2. GitHub 리포지토리 연결
# 3. 루트 디렉토리: frontend
# 4. 환경변수 설정 (setup-env.sh 출력값 사용)

# 방법 2: 로컬 개발
cd frontend
npm install
npm run dev
# http://localhost:3000
```

### **비용 최적화**
- **DSQL 사용**: RDS 대비 90% 비용 절감 (월 $40 → $1-5)
- **서버리스 아키텍처**: 사용량 기반 과금
- **S3 Lifecycle**: 7일 자동 삭제
- **Lambda 동시 실행 제한**: 10개

### **리소스 삭제 방법**
```bash
# 1. CloudFormation 스택 삭제
aws cloudformation delete-stack --stack-name love-q-backend-dev --region us-east-1

# 2. Amplify 앱 삭제 (콘솔 또는 CLI)
amplify delete

# 3. S3 버킷 수동 정리 (업로드된 파일이 있는 경우)
aws s3 rm s3://love-q-uploads-dev --recursive
```

### **주요 API 엔드포인트**
```http
POST /analyze-speech       # 말투 분석
POST /analyze             # AI 답변 생성  
POST /partner-profile     # 상대방 프로필 관리
POST /emotion-analysis    # 감정 분석
GET  /conversation-history # 대화 기록 조회
```

### **예상 사용 사례**

#### 1. **연애 초기 단계**
```
상황: 썸을 타는 상대에게 첫 고백을 준비하는 대학생
사용법: 
- 상대방의 성격과 대화 패턴 분석
- 간접적 소통을 선호하는 성향 파악
- 안전형 → 균형형으로 단계적 접근 전략 제안
```

#### 2. **장거리 연애**
```
상황: 연인과 멀리 떨어져 있어 텍스트 소통이 주가 되는 커플
사용법:
- 감정 상태 분석으로 상대방 기분 파악
- 위로나 격려가 필요한 타이밍 감지
- 각자의 말투에 맞는 표현 방식 제안
```

#### 3. **직장 내 관계**
```
상황: 호감을 가진 동료와의 적절한 선 유지
사용법:
- 업무 관계를 고려한 안전형 답변 중심
- 직장 예절을 지키면서도 관심 표현하는 방법
- 오해 소지가 없는 소통 방식 제안
```

#### 4. **재회 상황**
```
상황: 과거 연인과의 재연결 시도
사용법:
- 과거 대화 패턴 분석으로 선호 스타일 파악
- 신중한 접근이 필요한 상황 인식
- 점진적 관계 회복을 위한 단계별 소통 전략
```

#### 5. **세대 간 소통**
```
상황: 나이 차이가 나는 상대와의 연애
사용법:
- 세대별 소통 특성 분석
- 격식도와 친근함의 적절한 균형점 찾기
- 상대방 세대에 어울리는 표현 방식 학습
```
